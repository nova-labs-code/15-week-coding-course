Week 4: CSS Layouts

Welcome to Week 4! This week focuses on **layout techniques** to control how elements appear on your page, using display properties, Flexbox, and positioning. Each day is structured using “Learn → See → Do → Check” for ~30 minutes of hands-on practice.

Day 1: Block vs Inline Elements

Learn: HTML elements have default display types: block elements like <div> and <p> take up full width, while inline elements like <span> or <a> only take the width of their content. See:

<div>I am a block</div>
<span>I am inline</span><span>Me too</span>
Do: Create a page showing a block element followed by multiple inline elements and observe how they behave. Check: Open in a browser to see the difference in width and stacking behavior.

Day 2: Display Property

Learn: CSS display property can change how elements behave: block, inline, inline-block, or none to hide elements. See:

p {display: inline-block; background: #ddd;}
div {display: none;}
Do: Apply display:inline-block to paragraphs and try hiding a div with display:none. Check: Observe in the browser how layout changes and hidden elements disappear.

Day 3: Introduction to Flexbox

Learn: Flexbox is a CSS layout module for aligning and distributing space among items in a container. See:

<div class="flex-container">
  <div>Box 1</div>
  <div>Box 2</div>
</div>

<style>
.flex-container {display: flex; gap:10px;}
.flex-container div {background:#ddd; padding:20px;}
</style>
Do: Create a flex container with three boxes and experiment with spacing using gap. Check: Open in the browser to see boxes aligned horizontally and spaced evenly.

Day 4: Flex Direction & Justify Content

Learn: Flexbox allows control of direction and alignment. Use flex-direction: row | column and justify-content: flex-start | center | space-between. See:

.flex-container {display: flex; flex-direction: column; justify-content: center;}
Do: Change your flex container to column direction and experiment with justify-content options. Check: Verify in the browser how boxes stack vertically or align in different ways.

Day 5: Align Items & Align Self

Learn: Flexbox also aligns items vertically using align-items for all children and align-self for individual elements. See:

.flex-container {align-items: center;}
.flex-container div:nth-child(2) {align-self: flex-end;}
Do: Align all boxes in the center and change the second box to align differently. Check: Observe how the vertical alignment of elements changes in the browser.

Day 6: Position Property

Learn: CSS position controls element placement: static, relative, absolute, fixed, and sticky. See:

div.relative {position: relative; top: 10px; left: 20px;}
div.absolute {position: absolute; top: 50px; left: 50px;}
Do: Make one div relative and another absolute and move them around the page. Check: Open in the browser to see how position affects element placement.

Day 7: Review & Mini Layout Page

Learn: Combine all CSS layout skills to structure your profile page professionally: block vs inline elements, display adjustments, Flexbox containers with direction, justify-content, align-items, and basic positioning. Do: Recreate your profile page layout using Flexbox for your sections, align boxes properly, and experiment with positioning. Check: Open in a browser to ensure the layout is clean, elements are aligned, and the page looks organized. This completes Week 4 CSS layout fundamentals.
