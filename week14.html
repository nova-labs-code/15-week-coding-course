Week 14: JS DOM Events & Forms – Part 2

This week extends DOM events and forms into interactive, dynamic page features including lists, adding/removing items, toggling classes, and live previews.

Day 1: Dynamic List Addition

Learn: Add items to a list dynamically using inputs and buttons.

Do: Type an item in the input box and click "Add".

    Tip: Use createElement and appendChild instead of innerHTML to prevent overwriting existing items.
    Day 2: Removing List Items

    Learn: Remove items dynamically using JS.

    Do: Click "Remove Last" to delete the last item from your list.

    Tip: Check if lastChild exists before removing to avoid errors.
    Day 3: Toggling Classes

    Learn: Toggle CSS classes on elements to change appearance dynamically.

    Do: Click the box to activate/deactivate styling.

    Click me
    Tip: Toggling classes is useful for active states, highlighting, and interactive effects.
    Day 4: Live Input Preview

    Learn: Show input content live on the page as users type.

    Do: Type in the username input and watch the preview update instantly.

    Tip: Use input event for real-time updates, change for after typing ends.
    Day 5: Multiple Event Listeners

    Learn: Apply multiple effects on a single element with different events.

    Do: Hover over the box to see color change and revert on mouse leave.

    Tip: Multiple listeners allow complex, interactive behaviors without combining logic into one function.
    Day 6: Form Submission with JS

    Learn: Prevent default form submission to handle data via JavaScript.

    Do: Enter email and submit. JS alerts your input without refreshing the page.

    Tip: Always use e.preventDefault() to stop the page from reloading on form submit.
    Day 7: Review & Interactive Page

    Learn: Combine list management, live previews, event listeners, and form handling for a dynamic page.

    Do: Build a mini todo or signup page with add/remove, live preview, and alerts for empty fields.

    Check: Browser should reflect all updates instantly. Text cannot be selected or copied.
