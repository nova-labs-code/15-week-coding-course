Week 2: HTML Intermediate

Welcome to Week 2! This week we build on your basic HTML skills and focus on **semantic structure, grouping elements, styling with colors, lists with links/images, tables, and forms**. Each day uses the “Learn → See → Do → Check” method and is designed to take about 30–40 minutes.

Day 1: Semantic HTML & Sections

Learn: Semantic HTML tags give meaning to your content. Using them improves accessibility, SEO, and readability. Common tags include:

  • <header> – The top section of your page, usually with a title or navigation
  • <footer> – The bottom section, for copyright, contact info, or links
  • <main> – Contains the main content of your page
  • <section> – Groups related content
  • <article> – Represents a self-contained piece of content like a blog post

See:

<header>
  <h1>My Website</h1>
</header>
<main>
  <section>
    <h2>About Me</h2>
    <p>I am learning HTML and building web pages.</p>
  </section>
</main>
<footer>
  <p>Copyright 2026</p>
</footer>

Do: Reorganize your Week 1 profile page. Use <header> for your name, <main> for content, and <section> for each major part like About Me, Hobbies, or Contact.

Check: Open in a browser and verify that your page sections are clearly separated, and headings are easy to read. Try resizing the window to see how the layout behaves.

Tip: Using semantic tags is better than generic <div> because it helps screen readers and search engines understand your content.
Day 2: Divs & Spans

Learn: <div> and <span> are non-semantic containers. Use <div> for block-level grouping (full width) and <span> for inline grouping (inside text). They are essential for styling or scripting content.

See:

<div class="bio">
  <p>I enjoy coding and reading.</p>
</div>

<p>I like <span style="color:red;">JavaScript</span> for web development!</p>

Do:

  • Wrap each paragraph on your profile page inside a <div>.
  • Highlight a word or phrase using <span> and apply a different color or style.
  • Try nesting a <span> inside another <span> with a different style.

Check: Open in a browser and confirm that your colored text appears correctly and paragraphs are grouped visually.

Tip: Use descriptive class names for your <div> and <span> elements so you can easily style them later.
Day 3: Colors & Backgrounds

Learn: HTML elements can be styled inline with the style attribute. You can change text colors, backgrounds, and borders:

<p style="color:blue;">This text is blue</p>
<div style="background-color:#f0f0f0; padding:10px;">Light gray section</div>

Do:

  • Change the text color of headings to make them more visible.
  • Apply background colors to each section to differentiate content areas.
  • Experiment with contrasting colors for text and background to improve readability.

Check: Open in a browser. Ensure that text is readable and sections are visually distinct. Adjust colors if needed for clarity.

Tip: Use web-safe colors or HEX codes to ensure colors appear consistently across browsers.
Day 4: Lists with Links & Images

Learn: Lists can contain links and images. Use <ul> for unordered lists and <ol> for ordered lists. Each list item uses <li> and can include any inline content.

<ul>
  <li><a href="https://example.com">Example</a></li>
  <li><img src="icon.png" alt="Icon"> Coding</li>
</ul>

Do:

  • Add a small image next to each hobby in your hobbies list.
  • Make one item link to an external website related to your hobby.
  • Try nesting a sub-list inside one of your hobbies for more detail.

Check: Open in a browser to verify that images display correctly, links work, and nested lists appear properly.

Tip: Always include alt text for images for accessibility.
Day 5: Tables

Learn: Tables organize data into rows and columns. Use <table> for the table, <tr> for rows, <th> for headers, and <td> for cells.

<table border="1">
  <tr><th>Day</th><th>Task</th></tr>
  <tr><td>Monday</td><td>Code</td></tr>
  <tr><td>Tuesday</td><td>Study</td></tr>
</table>

Do: Create a table for your weekly schedule with at least 4 rows and 2 columns.

Check: Open in a browser and ensure the table aligns correctly, borders are visible, and text is readable.

Tip: Always use <th> for headers to improve accessibility and readability.
Day 6: Forms (Basic Inputs)

Learn: Forms collect user input. Use <form> with <input>, <textarea>, and <button>. Labels improve accessibility.

<form>
  <label>Name:</label><input type="text" placeholder="Your Name">
  <label>Email:</label><input type="email" placeholder="Your Email">
  <label>Message:</label><textarea placeholder="Type here"></textarea>
  <button type="submit">Submit</button>
</form>

Do: Add a contact form with Name, Email, and Message fields plus a submit button. Style labels and inputs for clarity.

Check: Open in a browser to ensure fields can be typed into and submit button appears clickable.

Tip: Always associate labels with inputs using the for attribute for accessibility.
Day 7: Review & Mini Intermediate Page

Learn: Combine all Week 2 skills to enhance your Week 1 profile page.

Do: Your enhanced page should include:

  • Semantic structure: <header>, <main>, <section>, <footer>
  • Grouped paragraphs with <div> and styled <span>
  • Colors and backgrounds for sections
  • Lists with images and links
  • A table showing your weekly routine
  • A contact form with labels, inputs, and submit button

Check: Open in a browser and ensure layout, readability, links, images, table, and form inputs all work correctly. Make adjustments to spacing, colors, or headings as needed.

Tip: Review your code for clean indentation and proper use of semantic tags.
